# FoldersModule

Иерархия папок с материализованными путями, поиск, мягкое удаление, кэширование размера.

## Эндпоинты

Все защищены `JwtAuthGuard`. Владелец определяется по `CurrentUser` из JWT.

| Метод | URL | Описание |
|---|---|---|
| POST | `/folders` | Создать папку |
| GET | `/folders/tree` | Дерево всех папок пользователя |
| GET | `/folders/search?q=...` | Поиск папок по имени |
| GET | `/folders/:id/contents` | Содержимое папки (папки + файлы) |
| GET | `/folders/:id/size` | Рекурсивный размер папки в байтах |
| PATCH | `/folders/:id` | Переименовать / переместить |
| DELETE | `/folders/:id` | Мягкое удаление (204) |

## FoldersService

### `create(ownerId, dto)`
Генерирует UUID заранее через `crypto.randomUUID()` — это нужно для построения пути до сохранения в БД. Если передан `parentId`, проверяет что родитель существует и принадлежит пользователю, проверяет глубину ≤ 10. Строит материализованный путь:
- корневая папка → `/{folderId}`
- вложенная папка → `{parent.path}/{folderId}`

### `getTree(ownerId)`
Загружает все не удалённые папки пользователя одним запросом, затем рекурсивно строит дерево в памяти через `buildTree()`. Возвращает `FolderTreeNodeDto[]`.

### `getContents(folderId, ownerId)`
Проверяет владение папкой, возвращает прямых потомков-папок. Поле `files` — пустой массив до реализации FilesModule.

### `update(id, ownerId, dto)`
Если `parentId` изменился — делегирует перемещение в `moveToNewParent()`. Переименование и перемещение можно делать одним запросом.
- `parentId: null` → переместить в корень
- `parentId: <uuid>` → переместить в другую папку
- `parentId` отсутствует → только переименование

### `softDelete(id, ownerId)`
Помечает `isDeleted = true` для самой папки и всех потомков. Потомки находятся через `path LIKE '{folder.path}/%'` — один UPDATE без рекурсии.

### `search(ownerId, query)`
Поиск по полю `name` через `ILIKE '%query%'`. Возвращает все совпадающие не удалённые папки пользователя.

### `getFolderSize(folderId, ownerId)`
Суммирует `size` всех файлов в папке и её потомках через SQL JOIN с таблицей `files`. Результат кэшируется в Redis под ключом `folder:size:{folderId}` на 5 минут.

### `moveToNewParent(folder, newParentId, ownerId)` (private)
Защищает от двух ситуаций:
1. Перемещение в себя или в собственное поддерево (проверка через `newParent.path.startsWith(folder.path + '/')`)
2. Превышение глубины 10 в целевом месте

Обновляет путь самой папки, затем находит всех потомков и обновляет их пути построчно.

### `buildTree(allFolders, parentId)` (private)
Рекурсивный метод. Фильтрует папки с нужным `parentId`, для каждой рекурсивно собирает детей. Работает на плоском массиве, загруженном одним запросом.

## Материализованные пути

```
/uuid-a                        ← корневая папка A
/uuid-a/uuid-b                 ← папка B внутри A
/uuid-a/uuid-b/uuid-c          ← папка C внутри B (глубина 3)
```

Преимущество: поиск всех потомков — одно условие `path LIKE '/uuid-a/%'`.

## Кэш в Redis

| Ключ | Значение | TTL |
|---|---|---|
| `folder:size:{folderId}` | размер в байтах (строка) | 300 сек |

Кэш инвалидируется при изменении файлов внутри папки (реализуется в FilesModule).
